## Style Rules

- **No em dashes (---) in any file in this project.** Use two regular dashes (--) or rephrase. This applies to all Markdown, Python, HTML, CSS, and configuration files.
- **No em dashes anywhere.** No `&mdash;`, no `&ndash;`, no literal Unicode em dash (U+2014).

## Project Goal

Build a modular Python pipeline for F1 aerodynamics analysis that connects first-principles analytical models to real FastF1 telemetry data. Output a standalone Mercedes-themed HTML portfolio site on GitHub Pages, with the same delivery style as the Soccer CFD and Airfoil CFD projects. Light SU2 CFD validation is included for the floor venturi tunnel ground effect simulation.

## Key Patterns (Follow Reference Projects)

- **Standalone HTML:** No build step. Pages in docs/ are self-contained. Nav sync via build_site.py.
- **Mercedes theme:** BG #1a1a1a, accent #00D2BE teal, secondary #d4d4d4, cards #2a2a2a.
- **Matplotlib style:** Use src/core/style.py for consistent dark theme on all plots.
- **Runner scripts:** One run_*.py per analysis page. idempotent. Output images to docs/assets/images/.
- **Component breakdown:** Use F1Car.component_breakdown() from models.py for downforce/drag distributions.
- **FastF1 compat:** numpy pinned to <2.0 in pyproject.toml (fastf1 uses np.NaN removed in numpy 2).

## FastF1 Critical Knowledge

- **Compatibility Fix:** numpy must be <2.0 (pinned to 1.26.4). FastF1 uses `np.NaN` removed in numpy 2.
- **Session loading:** `session = fastf1.get_session(year, gp, 'R')` then `session.load()`
- **Telemetry data:** `lap.get_telemetry()` returns DataFrame with Speed, RPM, Gear, Throttle, Brake, DRS, etc.
- **Track maps:** Use `track = session.get_track_status()` and FastF1 plotting utilities.
- **Caching:** FastF1 caches API responses by default in `~/.cache/fastf1/`.

## SU2 v8.4 Knowledge (Harrier, Rosetta x86_64)

- **Binary:** `/Users/ajeet/SU2_CFD/bin/SU2_CFD`
- **Config quirks for incompressible:** VISCOSITY_MODEL= CONSTANT_VISCOSITY, INC_NONDIM= INITIAL_VALUES (mu = 1/Re), CONV_NUM_METHOD_FLOW= FDS, MUSCL_FLOW= NO for RANS
- **Moving wall:** SURFACE_MOVEMENT= MOVING_WALL, MARKER_MOVING= (wall_name)
- **Mesh export:** Must create surface physical groups and call gmsh.model.mesh.createTopology() before gmsh.write()
- **Venturi setup:** 2D domain with profiled floor + diffuser on bottom, moving wall, symmetry top, ride height parameter, diffuser angle sweep (10-20 deg), velocity sweep (40-80 m/s)
