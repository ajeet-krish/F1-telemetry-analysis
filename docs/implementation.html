Code Architecture

Modular Python pipeline: analytical models, FastF1 telemetry, and visualization

Project Structure

The project is organized into three tiers: core models, analysis modules, and runner scripts. Each analysis module produces standalone visuals for the HTML portfolio site.

Core Models

The F1Car class in src/core/models.py combines front wing, rear wing, and floor models with body drag. Each component is a dataclass with calibrated coefficients: 

class FrontWing:
    # Multi-element inverted airfoil
    cl_alpha = 8.0  # per radian (with slot effect)
    cd0 = 0.025     # zero-lift drag
    k_induced = 0.12

class RearWing:
    cl_alpha = 8.5
    drs_drag_reduction = 0.70
    drs_downforce_loss = 0.50

class Floor:
    cl_base = -0.5
    # Ground effect: cl = cl_base * (1 + A/h + B/h^2)
    A = 0.05; B = 0.001

The component_breakdown() method returns downforce and drag per component, used by all analysis modules.

Telemetry Wrapper

TelemetryLoader in src/core/telemetry.py wraps FastF1 with a simple interface: 

loader = TelemetryLoader(2024, "Monaco", "R")
lap = loader.fastest_lap("VER")
tel = lap.get_telemetry()  # Speed, nGear, DRS, Throttle, etc.

FastF1 caches API responses in ~/.cache/fastf1/ for fast re-runs. Note: nGear (not Gear) is the column name for gear position.

Runner Pattern

Each analysis module has a matching runner script that imports and runs all visual functions: 

#!/usr/bin/env python3
"""Runner for downforce analysis."""
from src.analysis.downforce import run_all

if __name__ == "__main__":
    run_all()

Runners save images to docs/assets/images/ and are idempotent (safe to re-run).

Extending the Pipeline

To add a new analysis:

  1. Create src/analysis/new_module.py with a run_all() function
  2. Create run_new_module.py that calls run_all()
  3. Create docs/new_module.html referencing the images
  4. Run uv run python -m src.build_site to sync nav

The F1Car model can be extended with new components (diffuser, bargeboards, T-tray) by adding dataclass configs and implementing cl() and cd() methods.
